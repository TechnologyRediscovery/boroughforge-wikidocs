# Creating an event alert & adding it to the property page start and stop alert
The property event alert will display in an orange banner at the top of the page.
Property alert events are special events - when added to a property,case, or permit file will trigger a special flag to appear on the very top of the property profile; warning users of a condition of importance on the property. 
Property alert events can be any type of event and catergory. As of Jan 2024 a dedicated event type"Property Alerts" was created to house the alert events. 
1. Users create alert events in pairs ( event tools, event setup and categories, new event category,property alerts)
Step 1:
2. Begin by making a new event category that represents the END of the alert. The Stop alert catergory is a regulary category, this is NOT called an alert event.  ( do not check the alert event box for a stop- check for a start event)
Setp 2:
3. Create the event category that will trigger the property alert to appear . The category title shoud be chosen carefully since the title text is the only displayed text on the alert flag. 
	 a. When configuring the alert event, check the box "Alert Event" 
   b. In the Alert stop category field select from the drop down box the ending alert event category 				that was previously created. 
   c. Configure the rest of the event category as appropriate ( min rank to create - staff, view, read only - update staff ( or other choice) - 
   d. Icon hints - use Inspection-pass for a cleared check mark. use inspection_fail for an exclamation image. 
   
 4. To trigger the property alert, users then add the alert event to the property itself or any case 			or permit file living inside that property. When the property profile is viewed, the alert will 		appear on the top of the property profile.  Be sure to refresh property page to view alerts / updates. Alerts are time sensitive, if a stop alert must be recorded after the start.
